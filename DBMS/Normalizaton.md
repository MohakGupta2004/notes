- Insertion Anomaly: You can’t add new data because the database demands extra information you don’t have yet.
Like you don't have a primary, composite or any key to identify it. Then you can't add it.

- Deletion Anomaly: Let's say you have a customer table and deleting a customer can erase the orders the customer 
make. you delete something important which kills other useful data. You may want the orders to stay there because 
you may require it as a proof.

- Updation Anomaly: Modifying a data which cause unintentional errors.

Elimination of Data Redundancy: One of the main features of normalization is to eliminate the data redundancy that can occur in a database. Data redundancy refers to the repetition of data in different parts of the database. Normalization helps in reducing or eliminating this redundancy, which can improve the efficiency and consistency of the database. ^RMsD01yC

Features of Normalization:  ^oJONgoe1

Ensuring Data Consistency: Normalization helps in ensuring that the data in the database is consistent and accurate. By eliminating redundancy, normalization helps in preventing inconsistencies and contradictions that can arise due to different versions of the same data. ^3DA8cslo

Simplification of Data Management: Normalization simplifies the process of managing data in a database. By breaking down a complex data structure into simpler tables, normalization makes it easier to manage the data, update it, and retrieve it. ^AP3OZt58
